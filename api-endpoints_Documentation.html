get_players

Overview: The get_players endpoint retrieves a list of all active players currently in the game session, providing detailed information about each player.

HTTP Method: GET

Endpoint: /get_players

Headers:

  • Authorization: Required. A valid API token to authenticate the request.

Arguments: No additional arguments are required for this endpoint beyond the standard headers.

Return Type: list[rcon.types.EnrichedGetPlayersType]: This endpoint returns a JSON list of player objects, each containing detailed information about an individual player.

Response Schema: Each item in the returned list includes the following fields:

  • name: String. The player's name.
  • steam_id_64: String. The player's 64-bit Steam ID.
  • country: String. The player's country, if available.
  • steam_bans: Object. Information about the player's Steam bans, if any.
    • SteamId: String. The player's Steam ID.
    • VACBanned: Boolean. Whether the player has VAC bans.
    • EconomyBan: String. The status of any Steam Economy bans.
    • CommunityBanned: Boolean. Whether the player is banned from Steam Community.
    • NumberOfVACBans: Integer. The number of VAC bans.
    • DaysSinceLastBan: Integer. Days since the last ban was issued.
    • NumberOfGameBans: Integer. The number of game-specific bans.
  • profile: Object. Additional profile information.
    • Include subfields like id, created, sessions, total_playtime_seconds, etc., as applicable to your API's return data.
  • is_vip: Boolean. Indicates whether the player is marked as a VIP in the game.
